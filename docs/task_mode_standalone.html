How This Works

This is a demonstration of how chaos testing should work for LLM applications:

  1. Select a real-world failure scenario (not abstract chaos parameters)
  2. See the pipeline visualization showing where failures occur
  3. Get actionable insights on what failed and how to fix it
  4. Receive actual code snippets for implementing fixes

Note: This demo runs entirely in your browser. For full functionality with real LLM testing, you'd need the Node.js server version.
